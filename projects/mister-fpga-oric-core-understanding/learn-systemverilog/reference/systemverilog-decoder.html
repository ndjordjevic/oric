SystemVerilog decoder — every token in Oric.sv

This card lists every SystemVerilog construct, operator, and literal form that appears anywhere in Oric.sv, with its meaning and the lesson that teaches it. Keep it open beside the source: when you hit a symbol on any line and aren't sure what it does, find it here. If a line contains only tokens in these tables, you can decode that line in full. (This is syntax only — what a given signal does functionally is in 01a-Oric-sv-understanding.md.)

How to decode any line of Oric.sv:
  1. Where am I? Inside an always @(...) block (clocked/sequential) or out in the module body (continuous)? This decides what <= / = mean.
  2. What's the left-hand side? A signal being declared (wire/reg + width), or an existing signal being driven?
  3. Break the right-hand side into tokens and look each up below: operators, literals, slices, { }.
  4. Is it an instantiation? ModName inst ( ... ) — then it's ports (§7), not an expression.

Assignment & blocks

TokenMeaningLesson

wire x = expr;Declare + continuously drive in one line (same as assign).2
always @(posedge clk)Run the block once per rising clock edge (sequential/clocked logic).4
begin … endGroups multiple statements (SystemVerilog's { }-for-blocks).4
<= (inside always)Non-blocking: RHS read from old values; all updates land together at the edge.4

reg tmp; after beginBlock-local register, private to that always, still with memory.9
