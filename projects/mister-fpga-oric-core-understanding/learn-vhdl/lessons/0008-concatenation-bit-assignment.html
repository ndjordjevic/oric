Learn VHDL · Lesson 0008

Concatenation & individual bit assignment

Source: annotated/rtl/oricatmos.vhd, lines 423–432 (SD bus mux) and lines 803–816 (★ SECTION 16 — VIA port wiring). Prior: Lesson 7.

The win for today

You'll read VHDL's concatenation operator (same symbol as SystemVerilog's bitwise-AND, different job — context tells them apart) and see individual bits of a bus assigned one at a time.

1. & is concatenation in VHDL, not bitwise-AND

427sd_rd <= "00" & pravetz_sd_rd WHEN pravetz_mode = '1' ELSE md_sd_rd;
This is the sharpest single symbol-collision in the whole series: VHDL's & is the concatenation operator (glue two vectors end to end into a wider one), while VHDL's bitwise-AND is the word AND (Lesson 4). SystemVerilog does the exact opposite: & is bitwise-AND, and concatenation uses { }. Two languages, same symbol, opposite meanings — this is the one worth double-checking every time you see a bare & in either language until it's automatic.

Line 427 glues a fixed 2-bit "00" onto the front of the 2-bit pravetz_sd_rd, producing a 4-bit result matching sd_rd's width — the top two bits are hardwired to 0 (drives 2 and 3 are unavailable in Pravetz mode), the bottom two carry the real drive-select bits. The rest of the line is Lesson 4's WHEN/ELSE: in Pravetz mode the glued-together vector drives sd_rd (the SD-card read-request bus); otherwise the Microdisc controller's own md_sd_rd passes through — this is in fact one line from the same Section-6 controller-mux block Lesson 4 ended on.

SystemVerilog equivalent: {2'b00, pravetz_sd_rd} — same concatenation idea from SV Lesson 1, opposite operator choice.

2. Assigning one bit of a bus at a time: signal(index) <= value;

805via_pb_in(2 DOWNTO 0) <= via_pb_out(2 DOWNTO 0);
806via_pb_in(3) <= kbd_int;
807via_pb_in(4) <= via_pb_out(4);
808via_pb_in(5) <= 'Z';
809via_pb_in(6) <= via_pb_out(6);
810via_pb_in(7) <= via_pb_out(7);

Each line drives one bit (or a slice) of via_pb_in independently and continuously — this is legal because these are all separate, non-overlapping bits/ranges of the same 8-bit bus. Effectively this builds up the full 8-bit via_pb_in value bit-by-bit: bits 2-0 mirror the VIA's own output, bit 3 carries the keyboard-interrupt flag, bit 4 mirrors output again, bit 5 is tri-stated ('Z' — unused/not driven, same meaning as SV's 'Z), and bits 6-7 mirror output.

SystemVerilog would need concatenation to build one 8-bit value in a single assignment (assign via_pb_in = {via_pb_out[7], via_pb_out[6], 1'bZ, via_pb_out[4], kbd_int, via_pb_out[2:0]};) since SV doesn't usually assign to arbitrary bit-slices of a signal from several separate statements the way VHDL does here. VHDL allows driving different bit-ranges of the same signal from multiple separate concurrent statements, as long as the ranges never overlap.

3. Complex boolean expressions — same operators, more of them

803via_pa_in <= (((via_pa_joy_value AND via_pa_joy_mask) OR (via_pa_out_oe AND NOT via_pa_joy_mask) OR (via_pa_out AND NOT via_pa_out_oe AND NOT via_pa_joy_mask))) WHEN (via_cb2_out='0' AND psg_bdir='0') ELSE
804             (((via_pa_joy_value AND via_pa_joy_mask) OR (via_pa_in_from_psg AND via_pa_out_oe AND NOT via_pa_joy_mask) OR (via_pa_out AND NOT via_pa_out_oe AND NOT via_pa_joy_mask)));

No new syntax — just AND/OR/NOT (Lesson 4) combined more heavily than before, still following the value WHEN condition ELSE value shape. Reading a dense expression like this: work from the innermost parentheses outward, and recognize the repeated three-way OR-of-ANDs pattern (a classic multiplexer built from gates) rather than trying to hold the whole line in your head at once.

The cast of signals, so nothing on these two lines is a mystery token: the target via_pa_in is the VIA chip's port-A input byte, and it's built from via_pa_joy_value and via_pa_joy_mask (the joystick's data byte and a mask saying which bits the joystick claims), via_pa_out and via_pa_out_oe (the VIA's own output byte looped back, and its per-bit output-enable), and via_pa_in_from_psg (the sound chip's readback byte, seen connected in Lesson 5's psg port map). The WHEN condition via_cb2_out = '0' AND psg_bdir = '0' asks "is the PSG currently not being addressed?" — if it is being addressed, the second variant (line 804), which includes the PSG readback, is selected instead. The source's own -- comment block just above these lines says the same thing in prose — worth reading alongside.

Quick check

Q1. In VHDL, what does & do, and what's the equivalent SystemVerilog operator that looks identical but means something else?
Answer: VHDL's & concatenates two vectors into a wider one; SystemVerilog's & looks the same but performs bitwise-AND — completely different operations behind the same symbol.
Q2. Why is it legal for lines 805–810 to each assign a different part of via_pb_in separately?
Answer: because each line drives a different, non-overlapping bit or range of the same 8-bit signal — VHDL allows a bus to be built up from several concurrent statements as long as no two statements drive the same bit.
You now have this in hand: VHDL's & concatenation operator (and its collision with SV's bitwise-AND symbol), assigning individual bits/slices of a bus from separate statements, and the confidence to parse a dense multi-line boolean expression by working from the innermost parentheses out.
