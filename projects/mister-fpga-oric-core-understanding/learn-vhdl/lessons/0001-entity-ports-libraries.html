Libraries, ENTITY & PORT

Source: core/rtl/oricatmos.vhd, lines 45–182 (★ SECTION 2 — Library imports, ★ SECTION 3 — Entity). Companion: 01b-oricatmos-vhd-understanding.md. Assumes SystemVerilog Lessons 1–2 (see learn-systemverilog).

The win for today

You'll read a VHDL file's opening — library imports and the entity/port declaration — and map every piece directly onto the SystemVerilog module (...) syntax you already know.

1. LIBRARY / USE — VHDL's import statement

47LIBRARY ieee;
48USE ieee.std_logic_1164.ALL;
49USE ieee.numeric_std.all;

LIBRARY ieee; says "I'm going to use the standard IEEE library." Each USE line then imports a specific *package* from it: std_logic_1164 gives you the STD_LOGIC types (VHDL's basic wire/bit types), numeric_std gives you arithmetic on bit vectors (unsigned, TO_UNSIGNED, etc., seen in later lessons).

Exactly like import { something } from 'package' in JS, or from ieee.std_logic_1164 import * in Python — pulling named definitions into scope before you can use them.
VHDL is case-insensitive for keywords and identifiers (ENTITY, entity, and Entity are the same token) — this file mixes cases fairly freely (compare PORT MAP in Section 7 to port map in Section 12). Don't read meaning into capitalization; it's a style inconsistency, not a semantic difference.

2. Comments: -- instead of //

1-- ★ SECTION 1 — License & origin

VHDL line comments start with -- (two dashes) rather than SystemVerilog's //. Same idea, different character.

3. ENTITY ... PORT ( ... ) END; — VHDL's module boundary

55ENTITY oricatmos IS
56	PORT (
57		CLK_IN : IN STD_LOGIC;
58		RESET : IN STD_LOGIC;
59		key_pressed : IN STD_LOGIC;
61		key_code : IN STD_LOGIC_VECTOR(7 DOWNTO 0);
63		pravetz_layout : IN STD_LOGIC := '0';
64		K7_TAPEIN : IN STD_LOGIC;
65		K7_TAPEOUT : OUT STD_LOGIC;
181	);
182END;

ENTITY name IS PORT ( ... ); END; is VHDL's version of SystemVerilog's module name ( ... ); ... endmodule — it declares the component's *interface*, i.e. its pins, without yet saying how it works internally (that's the separate ARCHITECTURE block, Lesson 2).

SystemVerilog fuses the port list and the implementation into one module ... endmodule. VHDL deliberately splits them: ENTITY is the interface (the "what"), ARCHITECTURE is a separate block holding the implementation (the "how"). One entity can even have multiple different architectures — not used in this file, but the split exists for that reason.

Each port line reads: name : direction type; — for example CLK_IN : IN STD_LOGIC; declares an input pin named CLK_IN, one bit wide. IN/OUT correspond directly to SystemVerilog's input/output port directions (implicit via context in the `.sv` file's included port list).

4. STD_LOGIC vs. STD_LOGIC_VECTOR(n DOWNTO 0)

VHDL typeMeaningSystemVerilog equivalent
STD_LOGICa single wire/bitan unwidthed wire (1 bit)
STD_LOGIC_VECTOR(7 DOWNTO 0)an 8-bit bus, indexed high (7) to low (0)wire [7:0]
DOWNTO plays exactly the role of SystemVerilog's [msb:lsb] colon — (7 DOWNTO 0) means the same range as SV's [7:0], just spelled as a word instead of a symbol. VHDL also allows TO for low-to-high ranges, but this file consistently uses DOWNTO, matching the SV convention you already know.

5. A default value right in the port declaration

63pravetz_layout : IN STD_LOGIC := '0';

:= here assigns a *default* value an input port takes if the instantiating code doesn't explicitly connect it. '0' (single quotes) is a single-bit literal — VHDL uses single quotes for one bit, double quotes for a vector of bits (you'll see "00" for a 2-bit literal in later lessons).

Single quotes ('0', '1') vs. double quotes ("00", "10") is not a stylistic choice — it changes the type. A single-quoted literal is exactly one bit (STD_LOGIC); a double-quoted literal is a bit vector, even if it's only one character long conceptually.

Quick check

Q1. What VHDL construct plays the role of SystemVerilog's endmodule, and what comes right before it here?
Answer: END; closes the ENTITY block, coming right after the closing ) of the port list.
Q2. How many bits wide is a port declared STD_LOGIC_VECTOR(3 DOWNTO 0), and what's the SystemVerilog spelling of the same range?
Answer: 4 bits; in SystemVerilog it would be written [3:0].
Q3. Why does the entity alone (without an architecture) not tell you how the module actually behaves?
Answer: because VHDL deliberately separates the interface (ENTITY) from the implementation (ARCHITECTURE) — the entity only lists the pins and their types/directions, not any logic.
You now have this in hand: library imports, -- comments, the ENTITY ... PORT (...) END; shape as VHDL's counterpart to module ... endmodule, STD_LOGIC/STD_LOGIC_VECTOR(n DOWNTO 0) types, and single- vs. double-quoted bit literals.
