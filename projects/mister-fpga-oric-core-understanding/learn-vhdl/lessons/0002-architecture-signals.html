ARCHITECTURE & SIGNAL

Source: annotated/rtl/oricatmos.vhd, lines 184–321 (★ SECTION 4 — Architecture: internal signal declarations). Prior: Lesson 1.

The win for today

You'll recognize the ARCHITECTURE block as where all the real logic lives, and read SIGNAL declarations — VHDL's internal wiring, equivalent to SystemVerilog's wire/reg.

1. ARCHITECTURE name OF entity IS ... BEGIN ... END name;

189ARCHITECTURE RTL OF oricatmos IS
190
191	-- Gestion des resets
192	SIGNAL RESETn : STD_LOGIC;
...
395BEGIN
...
929END RTL;

This is the implementation body for the oricatmos entity from Lesson 1. RTL is just a name for this particular architecture (arbitrary — VHDL allows multiple named architectures per entity, though this design only defines one). Everything between ARCHITECTURE ... IS and BEGIN is the declarative region — signals, components — and everything between BEGIN and END RTL; is the actual logic.

There's no SystemVerilog equivalent to this two-part split (declare, then BEGIN, then logic) — in a module, declarations and logic can be interleaved freely. VHDL is stricter: all signals must be declared before the BEGIN that starts the architecture body.

2. SIGNAL — VHDL's internal wire

198SIGNAL cpu_ad : STD_LOGIC_VECTOR(23 DOWNTO 0);
201SIGNAL cpu_rw : STD_LOGIC;
194SIGNAL delay_count : STD_LOGIC_VECTOR(7 DOWNTO 0) := (OTHERS => '0');

SIGNAL name : type; declares an internal connection — something that isn't a port (not visible outside this entity) but carries a value between different pieces of logic inside the architecture, exactly like a SystemVerilog wire or reg declared inside a module.

A VHDL SIGNAL covers what SystemVerilog splits into wire and reg — the same keyword is used whether the signal ends up driven continuously (like a wire) or from inside a clocked PROCESS (like a reg). VHDL decides which based on how it's assigned later, not by a different declaration keyword.

Line 194 shows an initial-value aggregate: (OTHERS => '0') means "every bit of this vector starts at 0" — VHDL's way of writing a same-value fill across a whole bus without spelling out every bit.

Just like SystemVerilog's initial value on a reg (Lesson 4 of the SV series), this is the FPGA's power-up/reset value, not a "default if never assigned" placeholder the way a software variable's initializer works.

3. Naming as documentation: prefixes and comments

204-- VIA
205SIGNAL via_pa_out_oe : STD_LOGIC_VECTOR(7 DOWNTO 0);
227-- PSG
228SIGNAL psg_bdir : STD_LOGIC;

No new syntax here — just worth noticing the organizing convention: groups of related signals are clustered under a French-language -- comment naming the subsystem (via, psg, ula, disk controller signals prefixed md_ for Microdisc or pravetz_). This mirrors Oric.sv's own ★-section convention, just at a finer grain.

Quick check

Q1. What must come before the BEGIN keyword inside an ARCHITECTURE block?
Answer: all declarations for that architecture — SIGNALs and COMPONENTs (Lesson 3) — the declarative region.
Q2. Does VHDL use a different keyword to declare a signal that will later be driven by a clocked process, versus one driven continuously?
Answer: no — both use SIGNAL; the distinction (roughly matching SystemVerilog's wire/reg split) comes from how the signal is assigned later in the code, not from a different declaration keyword.
You now have this in hand: the ARCHITECTURE ... BEGIN ... END shape, SIGNAL as VHDL's unified wire/reg declaration, and the (OTHERS => '0') aggregate for initializing a whole bus.
